# CS-300: Analysis and Design Portfolio

This repository contains selected projects completed for **CS-300: Analysis and Design** at **Southern New Hampshire University**. These artifacts demonstrate my understanding of data structures, algorithm design, runtime analysis, and software development using C++.

## Repository Contents

### Project One

**Runtime Analysis and Data Structure Evaluation**

This document compares the runtime and memory characteristics of vectors, hash tables, and binary search trees for an academic advising application. The analysis evaluates the advantages and disadvantages of each data structure and justifies selecting a Binary Search Tree based on the project's functional requirements.

### Project Two

**ABCU Advising Assistance Program**

This project is a C++ application that reads course information from an input file, stores the data in a Binary Search Tree, and allows users to:

* Load course data from a file.
* Display all computer science courses in alphanumeric order.
* Search for an individual course.
* Display the course title and prerequisite information.

# Reflection

## What was the problem you were solving in the projects for this course?

The primary objective of these projects was to design and implement an advising assistance program for ABC University. The application needed to efficiently organize course information, display an alphabetically sorted list of courses, and allow advisors to quickly search for individual courses and their prerequisites. Accomplishing these requirements required selecting an appropriate data structure and implementing algorithms that balanced efficiency, readability, and maintainability.

## How did you approach the problem? Consider why data structures are important to understand.

I approached the project by first creating pseudocode and evaluating multiple data structures before writing any code. Comparing vectors, hash tables, and binary search trees demonstrated how the choice of data structure directly affects search performance, insertion efficiency, memory usage, and the complexity of implementation. This process reinforced the importance of selecting a data structure that aligns with the functional requirements rather than simply choosing the fastest option for one operation.

## How did you overcome any roadblocks you encountered while going through the activities or project?

Several challenges arose while implementing the Binary Search Tree, particularly with recursive insertion, searching, and validating prerequisite relationships while reading the input file. I addressed these issues by testing individual functions incrementally, debugging compiler and runtime errors, and verifying the program against the provided sample output. This iterative process helped identify and resolve issues before moving on to the next component.

## How has your work on this project expanded your approach to designing software and developing programs?

This course reinforced the importance of planning before implementation. Developing pseudocode, performing runtime analysis, and comparing alternative data structures provided a stronger understanding of algorithmic design before writing code. It also emphasized the value of testing throughout development rather than waiting until the end of the project.

## How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

These projects encouraged me to write code using modular functions with clearly defined responsibilities, descriptive variable names, and meaningful comments. Separating file processing, searching, traversal, and display logic into individual functions improved readability and simplified debugging. I now place greater emphasis on writing code that is easier to understand, maintain, and extend while following industry-standard programming practices.
